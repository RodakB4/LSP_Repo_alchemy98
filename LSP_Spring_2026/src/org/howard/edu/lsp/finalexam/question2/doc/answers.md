Design Explanation:

The Template Method pattern is implemented in the abstract Report class, where generateReport() is the template method marked final so subclasses cannot alter the workflow. The fixed sequence — loadData(), formatHeader(), formatBody(), formatFooter() — is defined once in Report.generateReport() and cannot be overridden. StudentReport and CourseReport each override only the four abstract hook methods to supply their specific data and formatting, while the overall structure stays controlled by the base class. This allows the Driver to treat both report types polymorphically through the List<Report> reference and call generateReport() on each without knowing the concrete type. Adding a new report type in the future requires only creating a new subclass and implementing the four hook methods, with no changes to the template or the driver.
